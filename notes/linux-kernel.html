Some Name Convention

  1. nr前缀（代表数目eg：nr_cpus 代表cpu的个数）。
  2. descr（描述符）
  3. processes （ tasks or threads）都代表进程。
  4. pf（page frame物理内存页）
  5. GFP（get free page 分配页框时使用的标志前缀）
  6. PG（struct page使用的标志前缀）
  7. idx（index）
  8. TLB（存储最近的线性地址与物理地址的映射，加快转换，是一种cache）

一些寄存器：

  1. idtr: IDT（中断描述符表）的地址，中断描述符表的第i项，对应于i号中段或者异 常（一共有256中中断）的描述符。
  2. Tr: tss（任务状态段）的选择子，以及TSS段的base，limit，tss的相关数据结构： tss_struct，init_tss（tss数组，每个cpu都有一个tss）
  3. gdtr: GDT（全局描述符表）的地址，GDT的每一项都是一个8字节的段描述符。包括 TSS的描述符和TLS（线程局部存储）的描述符，LDT的描述符，每一个cpu有一个GDT。
  4. Ldtr: LDT（局部描述附表）的地址
  5. Cr3: 保存当前进程的页全局目录（page global directory）的地址。

进程

内核为每一个进程都创建一个独立的内核栈，当进程在用户态执行时，内核会将该进程 的内核栈指针esp保存在TSS（即esp0）中，当进程执行系统调用进入内核态时，会从TSS 读取该进程的内核栈的指针esp0，同时将用户态的cs，eip等指针保存在内核栈中，当进 程要从内核态返回用户态时，它就会从内核栈读取保存的用户态寄存器的值，注意因为 每一个cpu只有一个TSS，所以TSS只保存当前运行进程内核栈的指针，当进程切换时TSS 的esp0的值会改变，所以当进程切换的时候，prev进程的esp0，esp，eip，fs，gs要保 存在task_struct的thread字段中，而一般用途的寄存器（eg：eax，ebx等）会保存在内 核栈中。因为进程切换发生在内核态，所以进入内核态之前用户态使用的寄存器已经保 存在该进程对应的内核栈中。
