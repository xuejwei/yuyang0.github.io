configuration

Linux(ubuntu, mint)

nvidia gpu

sudo apt-get install nvidia-cuda-toolkit nvidia-opencl-dev

restart the OS.

ATI GPU

  1. 安装依赖包 

    sudo apt-get install dh-modaliases execstack debhelper dkms lib32gcc1
  2. 安装驱动, 进入该网址 选择你的驱动类型，下载安装
  3. 到官网 安装APP SDK, CodeXL(debug tool).

windows(pyopencl)

  1. 安装Opencl, 这个比较简单，先装显卡驱动，在装APP SDK
  2. 先安装python环境
    • 安装Enthought Canopy
    • 接着是安装Numpy，boost_python, PyOpenCL这几个库，安装方法是去 http://www.lfd.uci.edu/~gohlke/pythonlibs/ 上下载文件，然后在命令行使用 pip install wheel—name 来安装，下载文件时注意python的版本一般选cp2.7以及 windows的版本，64位就选amd64
    • 需要IDE，那么就下载一个pycharm

DATETYPES

  1. 在kernel中，只能用C99的内置类型， 比如char，bool，int，float等等，但是这些 内置类型都是指定了确切大小的。
  2. 在host代码中，如果直接使用C语言的内置类型，那么类型的大小不确定，所以如果 host创建的数据是要传送到kernel的，那么这种不确定会造成问题，正是出于这种原 因 OpenCL定义了一系列以 cl 开头的类型，这种类型只能在host代码中使用， kernel中不能用，实际上在host代码中还有一种方法，就是使用C99的 stdint.h 提供的类型，比如 int32_t 就是4个字节.

图像类型中的坑

  1. 图像类型虽然支持CL_RGB 但是这个不支持每一个channel是8位的数据， 所以大多数 时候你必须给RGB的数据添加一位，来让它变成rgba，这主要是出于对齐考虑
  2. 图像类型不能同时读写，所以你创建图像buffer时必须指定是只读还是只写。 OpenCL2.0开始支持图像类型的可读可写.

Optimization

  1. work group 的大小最好是64的倍数
  2. 使用local memory可以显著的提高性能
