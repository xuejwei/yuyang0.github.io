Chp 1

  1. a programming language must have some primitive procedures and primitive data, for example mathmatics operations(+ - * / and so on),and the numbers(primitive data).
  2. Nesting of combinations provides a means of combining operations.
  3. Definitions that associate names with values provide a limited means of abstraction

提供基本的数据与函数，然后通过嵌套来构建组合式以及复合函数，同时为复合 函数或者组合式命名也给抽象带来了方便。

Some Concepts

  • 正则序(CBN)：在计算过程中先代换，完全代换后在一次求值
  • 应用序(CBV)：在计算过程中先求值在代换
  • 二者的比较：正则序相对与应用序一般会有较多的重复运算，因为过程体中可以多 次引用形参，那么正则序代换后就会多次求值，而应用序则是先求值在代换所以没 有这个问题，对于应用序，每个过程的形式参数都会先求值在进入过程计算
  • recursive procedure:语法上的递归
  • recursive process:计算过程的递归

语法上递归的函数可能表示的是一个递归计算过程，如果程序设计语言支持尾递归优 化，那么可以获得和循环相同的效率，本质上说，循环结构都只是语法糖衣，scheme 支持尾递归优化，C不支持尾递归优化。

  • 高阶函数： 函数可以作为参数传递，可以作为结果返回。

Chp2 (Data Abstraction)

当要构建一个抽象数据类型时,先应该想明白,数据类型应该提供哪些接口,这一般就是一 些procedure,这些procedure根据作用不同可以分为三类:

  1. Constructor: 用来构建该数据类型
  2. extractor: 用来从该数据类型中抽取信息
  3. predictor: 用来判断该数据类型的变种,比如说你要构建lambda calculus 表达式的 数据抽象,那么该数据类型要么是一个identifier,要么是一个 (lambda(x) body) 表 达式,那么你就需要提供两个predictor(identifer?, lambda-exp?)来判断到底属于 哪一种变种.

对于有理数而言:

  • make-rat : constructor 因为它是通过两个数来构建一个有理数
  • numer: extractor 它是从一个有理数中抽取出分子这个信息
  • denum: 而extractor 它是从一个有理数中抽取出分母这个信息

因为只有一个变种,所以没有predictor

Multiple Representation(2.4)

when multiple representations of a data type needs to coexist in a system. there are three ways to do this.

  1. Tagged Data : Every representation use a unique tag
  2. Data-Directed: Use a two dimensions(operations X type) table to dispatch the operation
  3. Message-Passing: Use the high order procedure to represent the data type.

primitive element

  • Numbers and arithmetic operations are primitive data and procedures.
  • Nesting of combinations provides a means of combining operations.
  • Definitions that associate names with values provide a limited means of abstraction.

some example in the book

GCD (greatest common divisors)

gcd(a, b) = gcd (b, r) (r = a % b)

证明：设a>b且 a = kb + r 其中k = a/b, r = a % b都为整数 若存在一个数m被a，b整除，则m也能被（kb + r）整除，又m能被kb整除， 所以必能被r整除

素数检查

  1. 在2～sqrt(x)之间遍历，如果找到一个因子则意味着x不是素数
  2. 费马小定律： 如果n是素数，则an % n = a % n (a是小于n的任意正整数）
  3. 通过费马小定律的检查只是素数的必要条件而不是充分条件，存在能通过所 有费马检查但却不是素数的数，但是这种数非常罕见，一般情况下对于数n如 果它不是素数，那么对于小于n的数中大部分不能通过费马检查，因此其概率 大于1/2，据此就可以构造概率算法

Chp3 (Modularity, Objects, and State)

A object is said to ``have state'' if its behavior is influenced by its history. We can characterize an object's state by one or more state variables, Since the states of objects in the system being modeled change over time, the state variables of the corresponding computational objects must also change. If we choose to model the flow of time in the system by the elapsed time in the computer, then we must have a way to construct computational objects whose behavior change as our program run.

object vs stream

object

计算机程序在某种程度上都是为了模拟外部的物理世界,为了使程序模块化,我们可以为 外部世界的每一个对象都构造一个计算对象,因为外部世界的对象有状态(state),所以我 们的计算对象也就需要状态(state variable),因为外部对象的状态会随着时间的流逝而 改变,所以我们的计算对象的状态也就需要改变,为此我们引入赋值表达式(assignment), 但是赋值的引入会产生令人困扰的复杂性,比如它使得替换模型失效.从而需要引入环境 模型,它使得程序必须非常注重语句的顺序,因为对象是可变的,在不同的地方你必须提供 该对象的正确版本,这就使得顺序变得很重要

stream

object中通过赋值来改变对象的状态,如果我们从整体上看就会发现,每一次赋值不过是 产生了流中的一个元素,但是流作为一个整体却是不变的,这样我们就可以去掉赋值

Chp4

任何一个程序都可以看做是一台抽象机器的描述,而解释器是一台特殊的机器,它能够模 拟其它机器的行为,因此解释器可以看作是一台 通用机器. 现在假设用lisp实现了一 个lisp 解释器, 那么这个解释器就可以模拟任何用lisp程序描述的机器,假设我现在给 该解释器送入一个lisp程序,而这个lisp程序实际是一个C语言的解释器,那么原来的lisp 解释器就可以模拟所有的用C语言描述的机器. 这里的深刻思想是: 任一解释器都可以 模拟其它的解释器, 因此原则上什么可以计算的概念就 与语言或者计算机无关. 通 用图灵机(元图灵机)实际就是一个解释器,它可以模拟图灵机的程序, 也就是说以一个图 灵机程序作为输入, 任何一个"有效过程" 都可以描述为图灵机的一个程序.
