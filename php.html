php 笔记

php是一门流行的web开发语言,以上手简单著称

变量

变量的命名与注意事项

  1. 变量必须要$符号，和perl类似.
  2. 常量用define定义，常量使用时不需要$符号，eg: define(“const\var”, 100),使用时直接用const\var不需要引号与$。
  3. 大小写：php的变量与常量区分大小写，而函数名，类名，方法名不区分大小写。

作用域

  1. 超全局变量
  2. 常量：一旦声明在脚本内全局可用，函数内外可用
  3. 全局变量，脚本内全局可用，但是函数内不可用，可用global关键词
  4. 函数内静态变量，和C语言相似
  5. 函数局部变量和C语言相似

操作符与控制结构

操作符

php的运算符基本和c语言类似。有一个===与==这个和js类似，一个类型与值都比较，一个只比较值。

控制结构

php的控制结构与C语言类似，if，switch，while，do…..while循环，for循环，多了一个foreach，foreach是专门用于数组的，break和continue的作用于C类似，还有一个exit可以直接结束php脚本的运行，执行错误检测时exit比较有用。

php数组

php数组是一个强大的数据结构

数组定义

  • $arr = array（’a’, ‘b’, ‘c’）
  • $arr = array(‘first’=>’a‘，‘second’ => ‘b’, ‘third’ =>’c’),

前一种的索引是0,1,2后者的索引是’first’,’second’,’third’，后一种方式和python的字典差不多。

循环访问

  1. foreach循环：foreach（$arr as $value）, foreach($arr as $key=>$value), 前一种方式只获得value，后一种获得key，value,foreach每次在运行之前都会将数组指针自动重置到数组开头，所以不需要使用reset函数。
  2. while(list($key, $value) = each($arr))， each：返回数组中当前的 键／值对并将数组指针向前移动一步，each返回的也是一个数组，list会将数组中索引为0,1,2…….对应的值依次赋给list的参数，注意索引只能是数字，不能是字符串，list的作用就是将数组的值赋给list指定的变量，注意：数组对应的key必须是0,1,2,3…………….,这种循环方式需要使用reset函数

数组排序（原地改变数组）

  • sort（对普通数组排序 ，不要应用于字典数组，因为它返回的排序数组会抹掉原字典数组的key，而将它的key变为0,1,2……）
  • asort（对字典数组的value排序）
  • ksort（对字典数组的key排序），

对应的反向（降序）排序rsort，arsort，krsort。Shuffle（随机排序），arrayreverse（反向排列）

其它数组函数

current（当前数组内部指针指向的元素，pos是current的别名） next（下一个元素的值） prev（前一个元素的值） end（内部指针重置到数组末尾，并返回末尾元素的值） reset（内部指针重置到开头，并返回开头元素的值 arraywalk（对数组的每个元素应用一个自定义函数，如果要改变数组本身，则自定义函数，必须传引用） count，sizeof（都返回数组的元素个数） arraycountvalues（返回数组元素中各个值出现的频率） extract（创建一系列以数组的key为名称的变量，而且这些变量的值是对应的value）。

字符串处理函数

不原地改变，而是返回一个新的结果字符串

常规字符串处理函数

  • trim(ltrim,rtrim(别名chop)：删除字符串的首尾两端的空格。ltrim只删除开头的空格，rtrim只删除结尾的空格。
  • nl2br: 将字符串中的\n转换为,<br />标签，这样可以在html中换行。
  • printf,sprintf:格式化输出。
  • strtolower（全部小写）,strtoupper（全部大写）,ucfirst（第一个字母大写）,ucwords（所有词的第一个字母大写）。
  • addslashes，stripslashes:将引号，反斜杠，NULL字符字符转义，以免在存入数据库时被当做控制字符，前者是添加，后者是删除，前者在存入数据库前调用，后者在从数据库取出数据后调用。
  • explode，implode（join）：前者分割字符串成数组，后者将数组合成字符串，都需要知道分隔符，join是implode的别名。
  • strtok：一次只会返回分割得到的一个字符串，而不是像explode那样返回数组
  • substr：指定开始与结尾来得到一个指定字符串的子字符串。
  • 字符串的比较：strcmp，strcasecmp（前者区分大小写，后者不区分，是字典序比较，故‘2‘>’12’）,strnatcmp,strnatcasecmp(按自然序比较，故’12’>’2’).
  • strlen：字符串的长度。
  • 查找子字符串：strstr（strchr 是strstr的别名，strstr区分大小写，stristr不区分大小写,都返回子字符串第一次出现的位置到结尾形成的字符串），strrchr(返回子字符串最后一次出现的位置到结尾形成的字符串)
  • 查找子字符串的位置：strpos，strrpos
  • 替换子字符串：str\replace，substr\replace

正则表达式函数

  • ereg（eregi不区分大小写）：搜索一个匹配，可以使用子模式。
  • ereg\replace,eregi\replace：替换
  • split：用指定的模式将字符串分割为一个数组

    以上的函数是POSIX正则表达式，而php5.3被废弃，全部采用perl兼容的正则表达式，主要函数如下：

  • Preg\match:搜索给定模式的一个匹配，可以将该匹配按子模式分割成数组返回，其中【0】是完整的匹配，【1】是第一个子模式的匹配，以此类推。
  • Preg\match\all：搜索所有给定模式的匹配，返回一个多维数组，该多维数组会包含所有的完整匹配以及子模式匹配，数组的组织形式由flag参数指定。
  • Preg\grep:搜索一个给定的数组
  • Preg\replace:替换。
  • Preg\split:分割为数组。 Perl正则表达式和POSISX表达式的区别：perl正则表达式的模式需要分隔符，分隔符放在pattern的开头和结尾，可以使/,#,~等，没有专门为区分大小写编写对应的函数，而是使用模式修饰符，模式修饰符放在pattern的分隔符的后面，主要有i（不区分大小写）s（点号匹配换行符）U逆转贪婪模式，perl从第一个合法匹配后停止，而且是贪婪模式。

文件

  • Fopen：打开文件，为了移植性模式应该加上b。可以打开网络上的远程文件。
  • Fwrite（fputs是fwrite的别名）：写文件
  • File\put\contents:可以在不使用fopen的情况下写入文件
  • Fgets,fgetss,fgetcsv:读取文件的一行数据，fgetss会去掉读出的行中的php与html标签，fgetcsv可以将读出的行分割为数组返回。
  • Readfile，file，fpassthru：读取整个文件。Readfile是向标准输出显示，file是读入一个数组。
  • Fgetc：读入一个字符，会显示EOF结束符，所以需要检查。
  • Fread：读入任意字节的字符串。
  • File\get\contents：在不使用fopen的情况下直接从文件读。
  • File\exists：文件是否存在。
  • Filesize：文件的大小。
  • Unlink：删除文件。
  • Rewind，fseek，ftell：rewind复位到文件开头，fseek文件中定位，ftell目前文件指针的位置。
  • Flock：文件锁定。

function funcname($a, $b….) { your code } 可以使用默认参数，方法是在形参后加=，eg：上例 $b = 1 函数使用return和C语言类似

MySQL

数据库设计的三大范式：

  • 第一范式：字段不可分，
  • 第二范式：有主键，非主键字段要依赖主键字段 只要数据列里面的内容出现重复，就意味着应该将表拆分，拆分形成的表必须用外键关联起来。
  • 第三范式：非主键字段不能相互依赖，也就是说表中的每一列必须与主键直接相关而不能间接相关。 与主键没有直接关系的数据列必须清除（创建一个表来存放他们）。

数据库的常用模式

主扩展模式

比如说公司有不同类型的员工，如设计师，程序员，PM等，那么可以先建一个员 工表，这个表存放所有类型的员工都有的属性，如姓名，性别，入职时间等等等， 然后在设计师一个表，程序员一个表，PM一个表，但是这些表都与员工表关联

主从模式

一张表的一条记录对应另一张表的多条记录。也就是一对多 eg：论坛的板块表与帖子表就是这种关系，而帖子表与回复表也是这种关系

名值模式
多对多模式

一本书可以有多个作者，一个作者也可以写多本书，所以书目信息表与作者信息 表就是多对多关系，那么设计时通常是在加入一个关联表，也就是通过3个表来 表示这种关系

其它技巧

  1. 对字符串可以使用单，双引号的场合尽量使用单引号，因为使用双引号效率不高。
  2. 对于一个没有设置的变量直接用if会有错误，因此应该使用isset或者empty检查，推荐empty。
  3. Showsource函数可以高亮显示程序代码。
  4. 密码加密推荐使用crypt函数的blowfish算法，不推荐使用sha1,或者md5.
  5. 生成一个唯一的字符串：uniqid函数。
  6. 对于表单，在点击提交按钮之前$POST中的索引是不存在的，所以这样调用$POST[‘name’]试图获得name输入框的值会报错，因为‘name’索引不存在。这时调用isset和empty返回false与true，如果点击了提交按钮，但是用户没有在输入框中输入任何数据，那么$POST[‘name’]的值是空字符串，这时调用isset会返回true，所以最好调用empty函数。
  7. 尽量使用字符串处理函数，而不要使用正则表达式，因为前者更快。
  8. _autoload($classname)需要自动加载的时候，定义这个函数。
  9. 每一个成功的POST页面都应该重定向，因为如果不重定向，那么刷新该页面会造成POST数据重复提交。。
  10. 函数返回引用时必须在定义与调用的时候都加上&，但是return语句不用 加&
